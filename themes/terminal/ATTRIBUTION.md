## Why vendor

This copy exists to apply small local fixes that the upstream project has not yet merged (e.g., a config-shape mismatch in the example site that prevented the logo partial from rendering). Vendoring keeps the blog buildable without waiting on upstream review cycles.

This is **not** a fork. It is a snapshot. Upstream contributions are still welcome at the source repo, and this directory is expected to be re-synced (not maintained as a long-lived divergence).
